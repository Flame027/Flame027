The Check Box Widget

View Class: CheckBox
Spec Class: CheckBoxSpec

The Check Box Widget has the same meaning like the Toggle Widget exept that it has a 'checked' sign as indication.
 
 

The Basics Section:

ID Unique symbolic name (ID) of the check box.  
This ID can be used by the application to access the the widget via "builder » componentAt:#ID".
Label Label of the check box.  
If the check box Label Is Image is turned off, this is the label string. Otherwise, it is the name of the selector, which returns the label - either a string or a bitmap from the application.
Model Aspect selector or binding for the state (on/off) of the check box.  
It should return a ValueModel* instance with a Boolean instance**.
Label Is Image Turns on/off that the name of the label is taken as selector. 
The selector should return a string or bitmap image used as logo in the check box.
Translate Label Turns on/off that the label string is translated via the resource mechanism to a national language string.  
A translation for that string must be define in the resource file of the application class. 
 
 

The Details Section:

Font Defines the font style of the check box's label. 
After selecting the check toggle the font style can be chosen by selecting one of the Family/Face/Style/Size buttons. If no font style is selected, the check box takes its default font style from the style sheet.
FG-Color Defines the foreground color of the check box, i.e. the color of the letters. 
After selecting the check toggle the foreground color can be chosen by selecting the desired colored button. If no color  is selected, the check box takes its default foreground color from the style sheet.
BG-Color Defines the background color of the check box, i.e. the color of the check box's view. 
After selecting the check toggle the background color can be chosen by selecting the desired colored button. If no color  is selected, the check box takes its default background color from the style sheet.
Initially Disabled Turns on/off that the check box is initially disabled.
Initially Invisible Turns on/off that the check box is initially invisible.
Can Tab Turns on/off that the check box can be reached by tabbing.
 
 

The Channels Section:

Enable Aspect selector or binding for en-/disabling the check box. 
It should return a 

    - Boolean instance** (only for static behavior) 
    - ValueModel* instance with a Boolean instance** 

A value model is used to provide a dynamically changing of the ability of the check box.
Label  Aspect selector or binding for the label of the check box. 
It should return a ValueModel* instance with a String instance. 
It is used to provide a dynamically changing of the label.
FG-Color Aspect selector or binding for the foreground color of the check box. 
It should return a 

    - Color instance  (only for static behavior) 
    - ValueModel* instance with a Color instance. 

A value model is used to provide a dynamically changing of the foreground color.
BG-Color Aspect selector or binding for the background color of the check box. 
It should return a 

    - Color instance  (only for static behavior) 
    - ValueModel* instance with a Color instance. 

A value model is used to provide a dynamically changing of the background color.
*  = ValueHolder, AspectAdaptor, BufferedValueHolder, RangeAdaptor, TriggerValue
** = true, false

 
Selected Name of the selector sent to the application when the check box was pressed.
